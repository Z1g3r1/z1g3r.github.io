Методы

45 мин Средний

Что такое методы?

Методы в Java — это блоки кода, которые выполняют определенную задачу и могут быть вызваны из других частей программы. Они позволяют организовать код в логические блоки, сделать его более читаемым и избежать повторения.

Синтаксис объявления метода

Метод состоит из следующих частей:

модификаторы типВозвращаемогоЗначения имяМетода(параметры) {
    // тело метода
    return значение; // если метод возвращает значение
}

Пример: Простой метод без параметров

public class Example {
    // Метод, который выводит приветствие
    public static void sayHello() {
        System.out.println("Привет, мир!");
    }
    
    public static void main(String[] args) {
        sayHello(); // вызов метода
    }
}

Типы методов

Методы без возвращаемого значения (void)

Используют ключевое слово void и не возвращают значение:

public static void printMessage(String message) {
    System.out.println(message);
}

Методы с возвращаемым значением

Возвращают значение указанного типа с помощью оператора return:

public static int add(int a, int b) {
    return a + b;
}

Статические и нестатические методы

Статические методы (с модификатором static) принадлежат классу, а не объекту, и могут быть вызваны без создания экземпляра класса.

Параметры методов

Методы могут принимать параметры — значения, которые передаются в метод при его вызове:

public static void greetUser(String name, int age) {
    System.out.println("Привет, " + name + "! Тебе " + age + " лет.");
}

Пример: Вызов метода с параметрами

public class MethodExample {
    public static void main(String[] args) {
        greetUser("Анна", 25); // Привет, Анна! Тебе 25 лет.
        
        int result = multiply(5, 7);
        System.out.println("Результат умножения: " + result); // 35
    }
    
    public static void greetUser(String name, int age) {
        System.out.println("Привет, " + name + "! Тебе " + age + " лет.");
    }
    
    public static int multiply(int a, int b) {
        return a * b;
    }
}

Перегрузка методов (Method Overloading)

Перегрузка позволяет создавать несколько методов с одинаковым именем, но разными параметрами:

public class Calculator {
    // Сложение двух целых чисел
    public static int add(int a, int b) {
        return a + b;
    }
    
    // Сложение трех целых чисел
    public static int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // Сложение двух чисел с плавающей точкой
    public static double add(double a, double b) {
        return a + b;
    }
    
    // Сложение массива чисел
    public static int add(int[] numbers) {
        int sum = 0;
        for (int num : numbers) {
            sum += num;
        }
        return sum;
    }
}

Компилятор определяет, какой метод вызвать, на основе количества и типов параметров.

Рекурсия

Рекурсия — это когда метод вызывает сам себя. Полезно для решения задач, которые можно разбить на подобные подзадачи:

public class RecursionExample {
    // Вычисление факториала числа рекурсивно
    public static int factorial(int n) {
        if (n == 0 || n == 1) {
            return 1; // базовый случай
        } else {
            return n * factorial(n - 1); // рекурсивный вызов
        }
    }
    
    // Вычисление чисел Фибоначчи
    public static int fibonacci(int n) {
        if (n <= 1) {
            return n;
        } else {
            return fibonacci(n - 1) + fibonacci(n - 2);
        }
    }
    
    public static void main(String[] args) {
        System.out.println("Факториал 5: " + factorial(5)); // 120
        System.out.println("Число Фибоначчи под номером 7: " + fibonacci(7)); // 13
    }
}

Модификаторы доступа

Модификаторы доступа определяют видимость методов и переменных:

Модификатор Видимость Описание
public Везде Метод доступен из любого другого класса
protected В пакете и подклассах Метод доступен в том же пакете и в подклассах
private Только в классе Метод доступен только внутри своего класса
(без модификатора) В пакете Метод доступен только в том же пакете

Передача параметров в методы

В Java параметры в методы передаются по значению. Это означает, что метод получает копию значения, а не оригинальную переменную:

public class ParameterPassing {
    public static void modifyPrimitive(int x) {
        x = x * 2; // изменяется копия, оригинал не меняется
        System.out.println("Внутри метода: " + x);
    }
    
    public static void modifyArray(int[] arr) {
        arr[0] = 999; // изменяется оригинальный массив
        System.out.println("Внутри метода: " + Arrays.toString(arr));
    }
    
    public static void main(String[] args) {
        int number = 5;
        System.out.println("До вызова: " + number);
        modifyPrimitive(number);
        System.out.println("После вызова: " + number); // останется 5
        
        int[] numbers = {1, 2, 3};
        System.out.println("До вызова: " + Arrays.toString(numbers));
        modifyArray(numbers);
        System.out.println("После вызова: " + Arrays.toString(numbers)); // изменится
    }
}

Методы с переменным числом параметров (Varargs)

Varargs позволяют передавать переменное количество аргументов одного типа:

public class VarargsExample {
    // Метод с переменным числом параметров
    public static int sum(int... numbers) {
        int total = 0;
        for (int num : numbers) {
            total += num;
        }
        return total;
    }
    
    // Можно комбинировать с обычными параметрами
    public static void printValues(String message, int... values) {
        System.out.print(message + ": ");
        for (int val : values) {
            System.out.print(val + " ");
        }
        System.out.println();
    }
    
    public static void main(String[] args) {
        System.out.println("Сумма: " + sum(1, 2, 3)); // 6
        System.out.println("Сумма: " + sum(1, 2, 3, 4, 5)); // 15
        System.out.println("Сумма: " + sum()); // 0
        
        printValues("Числа", 1, 2, 3, 4, 5);
    }
}

Пример программы

import java.util.Arrays;

public class MethodsExample {
    public static void main(String[] args) {
        // Вызов различных методов
        printWelcomeMessage();
        
        int result = add(15, 27);
        System.out.println("15 + 27 = " + result);
        
        double circleArea = calculateCircleArea(5.0);
        System.out.println("Площадь круга с радиусом 5: " + circleArea);
        
        int[] numbers = {7, 3, 9, 2, 5};
        int max = findMax(numbers);
        System.out.println("Максимальный элемент в массиве: " + max);
        
        // Перегруженные методы
        System.out.println("Сумма двух целых: " + add(10, 20));
        System.out.println("Сумма трех целых: " + add(10, 20, 30));
        System.out.println("Сумма двух вещественных: " + add(3.5, 2.7));
        
        // Рекурсивный метод
        int factorial = calculateFactorial(5);
        System.out.println("Факториал 5: " + factorial);
        
        // Метод с varargs
        int sum = calculateSum(1, 2, 3, 4, 5);
        System.out.println("Сумма чисел: " + sum);
    }
    
    // Метод без параметров и возвращаемого значения
    public static void printWelcomeMessage() {
        System.out.println("Добро пожаловать в программу работы с методами!");
    }
    
    // Метод с параметрами и возвращаемым значением
    public static int add(int a, int b) {
        return a + b;
    }
    
    // Перегруженный метод
    public static int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // Еще один перегруженный метод
    public static double add(double a, double b) {
        return a + b;
    }
    
    // Метод, вычисляющий площадь круга
    public static double calculateCircleArea(double radius) {
        return Math.PI * radius * radius;
    }
    
    // Метод для поиска максимального элемента в массиве
    public static int findMax(int[] array) {
        if (array.length == 0) {
            throw new IllegalArgumentException("Массив не может быть пустым");
        }
        
        int max = array[0];
        for (int i = 1; i < array.length; i++) {
            if (array[i] > max) {
                max = array[i];
            }
        }
        return max;
    }
    
    // Рекурсивный метод для вычисления факториала
    public static int calculateFactorial(int n) {
        if (n < 0) {
            throw new IllegalArgumentException("Число не может быть отрицательным");
        }
        if (n == 0 || n == 1) {
            return 1;
        }
        return n * calculateFactorial(n - 1);
    }
    
    // Метод с переменным числом параметров
    public static int calculateSum(int... numbers) {
        int sum = 0;
        for (int num : numbers) {
            sum += num;
        }
        return sum;
    }
}

Ключевые моменты

  • Методы позволяют организовать код в логические блоки и избежать повторения
  • Методы могут принимать параметры и возвращать значения
  • Перегрузка методов позволяет создавать несколько методов с одинаковым именем, но разными параметрами
  • Рекурсия — это когда метод вызывает сам себя
  • Модификаторы доступа определяют видимость методов
  • Параметры примитивных типов передаются по значению, а объекты — по ссылке
  • Varargs позволяют передавать переменное количество аргументов
  • Хорошие методы имеют понятное имя и выполняют одну четкую задачу
