Классы и объекты

50 мин Средний

Введение в объектно-ориентированное программирование (ООП)

Объектно-ориентированное программирование — это парадигма программирования, в которой программа представляется как набор объектов, взаимодействующих друг с другом. Java является полностью объектно-ориентированным языком.

Основные принципы ООП

Принцип Описание Преимущества
Инкапсуляция Объединение данных и методов в единый объект, скрытие внутренней реализации Повышение безопасности, упрощение использования
Наследование Создание новых классов на основе существующих с наследованием их свойств и методов Повторное использование кода, иерархическая организация
Полиморфизм Возможность объектов с одинаковой спецификацией иметь различную реализацию Гибкость, расширяемость кода
Абстракция Выделение основных характеристик объекта и игнорирование несущественных деталей Упрощение сложных систем, сокрытие сложности

Что такое классы и объекты?

Класс — это шаблон или чертеж для создания объектов. Он определяет свойства (поля) и поведение (методы) объектов.

Объект — это экземпляр класса, конкретная реализация, созданная на основе класса.

Аналогия: Класс — это чертеж дома, а объект — это конкретный дом, построенный по этому чертежу.

Создание класса в Java

Класс объявляется с помощью ключевого слова class:

public class Car {
    // Поля класса (свойства)
    String brand;
    String model;
    int year;
    double price;
    
    // Методы класса (поведение)
    void startEngine() {
        System.out.println("Двигатель запущен");
    }
    
    void stopEngine() {
        System.out.println("Двигатель остановлен");
    }
    
    void displayInfo() {
        System.out.println(brand + " " + model + " (" + year + ")");
    }
}

Создание объектов

Объекты создаются с помощью ключевого слова new:

public class Main {
    public static void main(String[] args) {
        // Создание объекта класса Car
        Car myCar = new Car();
        
        // Установка значений полей
        myCar.brand = "Toyota";
        myCar.model = "Camry";
        myCar.year = 2020;
        myCar.price = 25000.0;
        
        // Вызов методов объекта
        myCar.displayInfo(); // Toyota Camry (2020)
        myCar.startEngine(); // Двигатель запущен
    }
}

Конструкторы

Конструктор — это специальный метод, который вызывается при создании объекта. Он используется для инициализации полей объекта.

public class Car {
    String brand;
    String model;
    int year;
    double price;
    
    // Конструктор по умолчанию (без параметров)
    public Car() {
        brand = "Unknown";
        model = "Unknown";
        year = 2023;
        price = 0.0;
    }
    
    // Конструктор с параметрами
    public Car(String brand, String model, int year, double price) {
        this.brand = brand;
        this.model = model;
        this.year = year;
        this.price = price;
    }
    
    // Другие методы...
}

Ключевое слово this используется для обращения к полям текущего объекта, когда имена параметров совпадают с именами полей.

Инкапсуляция и модификаторы доступа

Инкапсуляция — это сокрытие внутренней реализации объекта и предоставление контролируемого доступа к данным через методы.

public class BankAccount {
    // Приватные поля (инкапсуляция)
    private String accountNumber;
    private double balance;
    
    // Конструктор
    public BankAccount(String accountNumber, double initialBalance) {
        this.accountNumber = accountNumber;
        this.balance = initialBalance;
    }
    
    // Публичные методы для доступа к полям (геттеры и сеттеры)
    public String getAccountNumber() {
        return accountNumber;
    }
    
    public double getBalance() {
        return balance;
    }
    
    // Методы для работы с балансом
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Средства зачислены: " + amount);
        } else {
            System.out.println("Неверная сумма для зачисления");
        }
    }
    
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println("Средства сняты: " + amount);
        } else {
            System.out.println("Недостаточно средств или неверная сумма");
        }
    }
}

Статические поля и методы

Статические поля и методы принадлежат классу, а не объекту. Они объявляются с ключевым словом static.

public class MathOperations {
    // Статическое поле
    public static final double PI = 3.14159;
    
    // Статический метод
    public static double calculateCircleArea(double radius) {
        return PI * radius * radius;
    }
    
    // Статический метод
    public static int add(int a, int b) {
        return a + b;
    }
}

// Использование статических методов
public class Main {
    public static void main(String[] args) {
        double area = MathOperations.calculateCircleArea(5.0);
        int sum = MathOperations.add(10, 20);
        
        System.out.println("Площадь круга: " + area);
        System.out.println("Сумма: " + sum);
        System.out.println("Значение PI: " + MathOperations.PI);
    }
}

Пакеты и импорт

Пакеты используются для организации классов в логические группы. Классы можно импортировать для использования в других классах.

Пример программы

// Класс Student
public class Student {
    private String name;
    private int age;
    private double gpa;
    private static int studentCount = 0;
    
    // Конструктор по умолчанию
    public Student() {
        studentCount++;
    }
    
    // Конструктор с параметрами
    public Student(String name, int age, double gpa) {
        this.name = name;
        this.age = age;
        this.gpa = gpa;
        studentCount++;
    }
    
    // Геттеры и сеттеры
    public String getName() {
        return name;
    }
    
    public void setName(String name) {
        this.name = name;
    }
    
    public int getAge() {
        return age;
    }
    
    public void setAge(int age) {
        if (age > 0) {
            this.age = age;
        }
    }
    
    public double getGpa() {
        return gpa;
    }
    
    public void setGpa(double gpa) {
        if (gpa >= 0 && gpa <= 4.0) {
            this.gpa = gpa;
        }
    }
    
    // Статический метод
    public static int getStudentCount() {
        return studentCount;
    }
    
    // Метод для отображения информации
    public void displayInfo() {
        System.out.println("Студент: " + name + ", Возраст: " + age + ", Средний балл: " + gpa);
    }
}

// Главный класс
public class SchoolManagement {
    public static void main(String[] args) {
        // Создание объектов Student
        Student student1 = new Student("Иван Петров", 20, 3.8);
        Student student2 = new Student("Мария Сидорова", 19, 4.0);
        Student student3 = new Student("Алексей Иванов", 21, 3.5);
        
        // Использование методов объектов
        student1.displayInfo();
        student2.displayInfo();
        student3.displayInfo();
        
        // Изменение свойств через сеттеры
        student1.setGpa(3.9);
        System.out.println("Новый средний балл Ивана: " + student1.getGpa());
        
        // Использование статического метода
        System.out.println("Всего студентов: " + Student.getStudentCount());
        
        // Демонстрация инкапсуляции
        // student1.name = "Новое имя"; // Ошибка! Поле private
        student1.setName("Иван Новый"); // Правильный способ
        System.out.println("Новое имя: " + student1.getName());
    }
}

Ключевые моменты

  • Классы — это шаблоны для создания объектов, определяющие их свойства и поведение
  • Объекты — это экземпляры классов, созданные на основе этих шаблонов
  • Конструкторы используются для инициализации объектов при их создании
  • Инкапсуляция позволяет скрыть внутреннюю реализацию и предоставить контролируемый доступ к данным
  • Геттеры и сеттеры используются для доступа к приватным полям класса
  • Статические поля и методы принадлежат классу, а не объекту
  • Ключевое слово this ссылается на текущий объект
  • Пакеты помогают организовать код в логические группы
  • ООП основывается на четырех принципах: инкапсуляция, наследование, полиморфизм и абстракция
